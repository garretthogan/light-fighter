# Project Instructions

## Resources

- For all problems related to three.js consult the source code in the parent directory at "/three.js/"
- Use plain javascript organized in thin hierarchies and patterns instead of intricate frameworks.
- Use classes to logically organize code into encapsulated objects that are conducive to understanding the repository
- Use the design system in the parent directory located at "/design-system/" for problems related to aesthetics and style and UX design.
- Clean up unused code. Mitigate tech debt when possible without causing regressions
- Good code is self documenting. Prefer simple to read code over verbose comments.
